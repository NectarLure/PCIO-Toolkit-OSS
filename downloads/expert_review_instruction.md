# PCIO Expert Review Portal 专家填写说明

感谢参与 PCIO Toolkit 的 DSR 专家形成性评审。本评审旨在检查人工制品的设计合理性、模块结构、场景适配性和证据边界清晰性，不用于生成任何虚构评价结果。

## 一、如何查看网站

可以直接打开：

```text
review_portal/index.html
```

也可以在 `review_portal` 目录下启动本地静态服务：

```bash
python -m http.server 8080
```

然后访问：

```text
http://localhost:8080
```

建议按以下顺序浏览：

1. `index.html`：了解研究目的、DSR定位和证据边界。
2. `artifact_overview.html`：审阅 PCIO 四层模型、模块结构和设计原则。
3. `module_demo.html`：查看模块输出结构，不涉及原始企业数据。
4. `evidence_package.html`：区分自动化测试、合成案例和真实 T0 案例。
5. `case_p.html`：审阅流程生产型企业 T0 基线诊断。
6. `case_t.html`：审阅罐区储运运营场景 T0 基线诊断。
7. `expert_review.html`：填写网页评分表，或下载 Markdown/CSV 评分表离线填写。

## 二、如何评分

请使用 1—5 分 Likert 量表：

- 1 = 非常不同意 / 明显不足
- 2 = 不同意 / 存在较多不足
- 3 = 一般 / 基本可接受但需要修改
- 4 = 同意 / 较合理
- 5 = 非常同意 / 很合理

评分维度包括：

- PCIO四层模型合理性；
- 题项覆盖性；
- 风险规则合理性；
- KPI建议可执行性；
- ROI情景表达谨慎性；
- 报告可理解性；
- 流程生产企业适配性；
- 罐区储运场景适配性；
- 证据边界清晰性；
- 整体实用潜力。

每项评分后建议写一句理由或修改建议。若无法判断，可在评论中说明需要补充什么材料。

## 三、如何提交反馈

可选择以下任一种方式：

1. 在 `expert_review.html` 中填写后，使用浏览器“打印”或“另存为 PDF”。
2. 下载并填写 `downloads/expert_review_form.md`。
3. 下载并填写 `downloads/expert_review_form.csv`，便于后续人工汇总。

本门户不会自动保存或上传专家输入。请通过研究团队指定的邮件、网盘或线下方式提交反馈文件。若没有指定提交方式，请先联系研究团队确认，不要把包含专家身份信息的文件上传到公开仓库。

## 四、评价边界

请特别注意：

- 真实案例仅为 T0 基线诊断案例，可用于评价基线诊断能力、场景适配性、风险识别、KPI建议和PDCA后续计划。
- 真实案例不能用于证明部署后改善、因果效果、真实ROI实现或长期运行稳定性。
- 合成案例仅用于端到端流程测试、规则响应检查和输出差异化演示。
- 自动化测试仅支持技术正确性、功能完整性、部署可行性和模块运行一致性。
- ROI 模块仅用于情景估算和方案比较，不代表真实收益验证。
- 当前没有专家评分结果、访谈结果、T1/T2 复测数据或客观绩效改善结果。

## 五、隐私提醒

请勿在反馈中写入真实企业名称、员工姓名、手机号、微信、邮箱、客户、供应商、合同号、项目名、精确地址、真实设备编号、车牌或罐区安防细节。如需举例，请使用 Case P、Case T、设备M01、Area 1、Line 1 等通用表达。
